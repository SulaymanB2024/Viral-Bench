# Viral-Bench public discovery matrix — 2026-07-18

## Purpose and boundary

This is a public-evidence discovery map for the acquisition-control program. It
is not a source-registry mutation, collection authorization, provider request,
or claim that a public account is active. Every candidate remains pending the
normal registry review. The corresponding machine-readable catalog has 40
unique seed identities: 28 internship/early-career (70%) and 12 broader
job-search (30%).

The company pages and public social destinations below are the current
official-source evidence recorded by the 2026-07-16/18 research checkpoint.
They establish a relevant public discovery lead, not content ownership,
performance, or permission to reuse creative material.

## Manifest interpretation

`internship-discovery-acquisition-draft-v1-20260718.json` uses deterministic
seed IDs and canonical handle, hashtag, URL, and query identities. It has zero
external-call authority and zero registry-mutation authority. Expected-yield
fields are capacity hypotheses (not outcomes): profile routes use a conservative
1–6 public records per seed; trend routes 1–5; query/hashtag/adjacency/audio
routes 1–10. Dedupe happens before quota measurement, and a missing HTTPS
evidence URL or timestamp is a build failure.

The initial route lineage is intentionally broad enough to find unique relevant
public evidence, but it does not weaken the existing normalized-post or viral
library schemas. A candidate profile must be accepted into the reviewed source
registry before it can enter any provider-backed profile route.
